# PharmaChain 自动化测试

## 📋 测试内容

此自动化测试脚本完整测试 PharmaChain 系统的工作流程：

### 1️⃣ Admin 部分
- ✅ Admin 登录
- ✅ 注册药品类别

### 2️⃣ Manufacturer 部分
- ✅ Manufacturer 登录
- ✅ 注册新药品（选择类别，上传PDF）
- ✅ 检查审批后获得的 MAL 号码
- ✅ 创建生产批次
- ✅ 检查库存增加状态
- ✅ 转移药品给 Retailer
- ✅ 检查库存扣除状态

### 3️⃣ 审批流程
- ✅ Admin 审批药品申请（生成 MAL 号码）
- ✅ Admin 审批批次申请（添加到库存）

### 4️⃣ Retailer 部分
- ✅ Retailer 登录
- ✅ 检查库存状态

### 5️⃣ QR Code 部分
- ✅ 生成二维码信息
- ✅ 扫描二维码追溯

---

## 🚀 快速开始

### 1. 安装依赖

```bash
cd auto
npm install
```

### 2. 确保后端服务器运行

```bash
# 在项目根目录
./start.sh
```

### 3. 运行自动化测试

```bash
cd auto
npm test
```

或者直接：

```bash
node test-complete-workflow.js
```

## 📝 测试数据

每次测试运行都会生成唯一的测试数据：

- 类别名称: `测试类别_<时间戳>`
- 药品名称: `测试药品_阿莫西林`
- 注册号: `REG-<时间戳>`
- 批次号: `BATCH-<时间戳>`
- 数量: 1000 单位
- 转移数量: 100 单位

## ✨ 验证点

每个测试步骤都验证以下内容：

1. **HTTP 状态码**: 确保API调用成功
2. **返回数据**: 验证响应数据结构正确
3. **业务逻辑**: 验证业务流程符合预期
4. **数据一致性**: 验证数据在各步骤间保持一致

## 🛠️ 故障排除

### 测试失败

如果测试失败，检查：

1. **后端服务器是否运行**:
   ```bash
   curl http://localhost:5000
   ```

2. **用户是否已创建**:
   ```bash
   cat backend/models/users.json
   ```

3. **端口是否正确**:
   - 后端: 5000
   - 前端: 3000

### 依赖问题

如果遇到 `Cannot find module 'form-data'`：

```bash
cd auto
npm install
```
